    Modelaci√≥n del Hidatoüîó

    Para representar el tablero del juego Hidato se utiliz√≥ una tupla de 3 elementos (nRows, nCols, setCells). En lo adelante, cuando se haga referencia a un tablero de Hidato se har√° referencia a la tupla (nRows, nCols, setCells).

    nRows: x ‚àà Œù. Representa la cantidad de filas del tablero

    nCols: x ‚àà Œù. Representa la cantidad de columnas del tablero

    setCells: El conjunto S de todos los vectores ‚å©x, y, v‚å™ donde x, y ‚àà Œù, 1 ‚â§ x ‚â§ nRows, 1 ‚â§ y ‚â§ nCols, v ‚àà Œñ, -1 ‚â§ v, ‚àÄ ‚å©x1, y1‚å™ ‚àà ([1, nRows]√ó[1, nCols]) ‚à© Z x Z el conjunto S contiene un √∫nico vector ‚å©x2, y2, v2‚å™ tal que x1 = x2 y y1 = y2, no existen 2 vectores ‚å©x1, y1, v1‚å™ ‚àà S y ‚å©x2, y2, v2‚å™ ‚àà S tal que x1 ‚â† x2, y1 ‚â† y2, v1 > 0, v2 > 0 y v1 = v2; y v ‚â§ nRows * nCols - |{‚å©x1, y1, v1‚å™ | ‚å©x1, y1, v1‚å™ ‚àà S ‚àß v1 = -1}|. Conjunto de vectores donde cada uno representa una celda en el tablero. La primera y segunda componente del vector representan la fila y columna respectivamente en la que se encuentra la celda dentro del tablero y la tercera componente representa el valor de la celda en el tablero. Estos valores son todos mayores iguales a -1 y representan lo siguiente:

    • -1 representa un obst√°culo, esa celda no se utiliza para el juego, de esta manera se pueden representar juegos de Hidato de la forma que se quiera utilizando obst√°culos (Externamente para el usuario se representa con 'x')
    • 0 Representa una celda en blanco, la cual debe ser rellenada para solucionar el Hidato (Externamente para el usuario se representa con '.')
    • Mayor que 0 representa una celda con un n√∫mero rellenado ya sea porque estaba as√≠ en un inicio o porque fue rellenado durante el proceso de resoluci√≥n del Hidato.

    Se define como una plantilla de Hidato a un tablero donde setCells no contenga vectores con valores mayores que 1 en su tercera componente.

    Se define como una plantilla de Hidato v√°lida a una plantilla de Hidato si partiendo desde ella y realizando transformaciones v√°lidas al conjunto setCells es posible llegar a un Hidato resuelto.

    Las transformaciones v√°lidas son aquellas que a partir de un tablero de Hidato se escoge un vector del conjunto setCells tal que su tercera componente sea igual a 0 y se modifica esa componente colocando un valor diferente de 0 siempre y cuando el resultado sea un tablero de Hidato.

    Se define que 2 vectores ‚å©x1, y1, v1‚å™ y ‚å©x2, y2, v2‚å™ del conjunto setCells de un tablero de Hidato son adyacentes si existe ‚å©x3, y3‚å™ ‚àà ([-1, 1]√ó[-1, 1]) ‚à© Z x Z - {‚å©0, 0‚å™} tal que ‚å©x1, y1‚å™=‚å©x2, y2‚å™+‚å©x3, y3‚å™.

    Se define como un Hidato resuelto a un tablero de Hidato que no contenga vectores en setCells con la tercera componente igual a 0 y que ‚àÄ ‚å©x1, y1, v1‚å™ ‚àà setCells con v1 > 1 existe un vector ‚å©x2, y2, v2‚å™ ‚àà setCells que es adyacente con ‚å©x1, y1, v1‚å™ donde v1 = v2 + 1.

    Algoritmo de Soluci√≥nüîó

    La idea general del algoritmo de soluci√≥n es utilizar un recorrido por el grafo de posibles transformaciones v√°lidas utilizando el algoritmo de b√∫squeda en profundidad o mejor conocido como DFS.

    La funci√≥n solve recibe un tablero, un paso y un l√≠mite (este l√≠mite es igual al mayor n√∫mero posible dentro del tablero o sea el mayor v) y realiza lo siguiente:

    • Si el paso es igual al l√≠mite m√°s uno es que el tablero est√° rellenado completo y por tanto se resolvi√≥ completamente, por lo que devuelve una lista conteniendo al tablero.
    • Si no, devuelve una lista resultante de concatenar las listas de tableros resultantes del proceso de utilizar cada tablero generado por la invocaci√≥n de la funci√≥n stepMatrix pas√°ndole los argumentos del tablero actual y el paso actual como argumento de un llamado recursivo a la funci√≥n solve con el paso siguiente y el mismo l√≠mite.

    La funci√≥n stepMatrix recibe un tablero y un paso y realiza lo siguiente:

    • Si hay alguna celda con el valor igual al paso actual dentro del tablero actual y bien ese valor es 1 o la celda es adyacente con la celda que tiene el valor del paso anterior dentro del tablero actual devuelve una lista conteniendo al tablero actual.
    • Si hay alguna celda con el valor igual al paso actual dentro del tablero actual y el valor NO es 1 y la celda NO es adyacente con la celda que tiene el valor del paso anterior dentro del tablero actual devuelve una lista vac√≠a.
    • Si no, devuelve una lista que contiene a los tableros resultantes del proceso de recorrer cada celda adyacente (que su valor sea diferente de 0) de la celda que tiene el valor del paso anterior dentro del tablero generando por cada una de estas celdas adyacentes un nuevo tablero con el valor de esa celda igual al paso actual.

    Ejemplo de pseudo implementaci√≥n en Python del algoritmo de soluci√≥n.

    def solve(table, step, limit):
    if step == limit + 1:
        return [table]
    else:
        return [
            item
            for new_table in step_matrix(table, step)
            for item in solve(new_table, step + 1, limit)
        ]

def step_matrix(table, step):
    if cell_in_table(step, table):
        if get_cell_value(get_cell(table, step)) == 1 or is_adyacents(get_cell(table, step), get_cell(table, step - 1)):
            return [table]
        else:
            return []
    else:
        return [
            update_cell_in_table(table, update_value_in_cell(step, cell))
            for cell in get_adyacents(table, get_cell(table, step - 1))
            if get_cell_value(cell) == 0
        ]

    Algoritmo de Generaci√≥nüîó

    La idea general del algoritmo de generaci√≥n es generar una plantilla de Hidato v√°lida aleatoriamente a partir de una cantidad de filas, de columnas y de un porcentaje de obst√°culos, resolver esta plantilla para obtener un Hidato resuelto y luego realizar un proceso de eliminaci√≥n de una cantidad de celdas (seg√∫n la dificultad seleccionada) cuyos valores sean mayores que 1 y menores que el m√°ximo v en un orden aleatorio comprobando en cada paso que al eliminar no sea posible generar m√°s de un Hidato resuelto (siendo el √∫nico posible, el Hidato resuelto desde donde se comenz√≥ el proceso de eliminaci√≥n).

    • La funci√≥n generate recibe el n√∫mero de filas, de columnas, el porcentaje de obst√°culos y la dificultad y realiza lo siguiente: Utilizando la funci√≥n generateHidatoSolved genera un Hidato resuelto que utiliza como argumento al llamar a la funci√≥n removeCells pas√°ndole adem√°s como argumento a esta una permutaci√≥n aleatoria de las casillas del Hidato resuelto (generado por la funci√≥n generateHidatoSolved) que se pueden eliminar (aquellas que el valor sea mayor que 1 y menor que el m√°ximo valor) y el n√∫mero de casillas que se puede eliminar seg√∫n la dificultad devolviendo finalmente la plantilla de Hidato.
    • La funci√≥n generateHidatoSolved recibe el n√∫mero de filas, de columnas y el porcentaje de obst√°culos y realiza lo siguiente: Comprueba si la pseudo plantilla de Hidato generada aleatoriamente por la funci√≥n generateRandomPseudoTemplate es una plantilla de Hidato v√°lida (que es posible a partir de ella llegar a un Hidato resuelto), si lo es, devuelve esa plantilla, si no, lo intenta nuevamente.
    • La funci√≥n generateRandomPseudoTemplate recibe el n√∫mero de filas, de columnas y el porcentaje de obst√°culos y realiza lo siguiente: Devuelve una pseudo plantilla (plantilla que no se ha comprobado que sea una plantilla de Hidato v√°lida) con la cantidad de filas y columnas recibida como argumentos y con una cantidad de casillas (seg√∫n el porcentaje de obst√°culos) marcadas como obst√°culos aleatoriamente.
    • La funci√≥n removeCells recibe una plantilla, una permutaci√≥n aleatoria de las casillas que se pueden eliminar y el n√∫mero de casillas que se deben eliminar seg√∫n la dificultad y realiza lo siguiente: Si la permutaci√≥n aleatoria de las casillas que se pueden eliminar es vac√≠a o el n√∫mero de casillas que se debe eliminar seg√∫n la dificultad es negativo se devuelve la plantilla si no se genera una nueva plantilla modificando el valor (poni√©ndolo en 0) de la casilla correspondiente (seg√∫n la permutaci√≥n aleatoria de las casillas) a partir de la plantilla recibida como argumento, con esta nueva plantilla comprueba si se generan m√°s de un Hidato resuelto, en caso de ser as√≠ devuelve el resultado de aplicar un llamado recursivo a removeCells pas√°ndole la plantilla recibida como argumento, la cola de la permutaci√≥n aleatoria de las casillas que se pueden eliminar y el mismo n√∫mero de casillas que se deben eliminar (pues en ese paso no es posible eliminar la casilla porque genera m√°s de un Hidato resuelto), en caso de generar solamente un Hidato resuelto devuelve el resultado de aplicar un llamado recursivo a removeCells pas√°ndole la nueva plantilla, la cola de la permutaci√≥n aleatoria de las casillas que se pueden eliminar y un n√∫mero menos de casillas que se deben eliminar (pues en ese paso es posible eliminar la casilla porque genera solamente un Hidato resuelto).

    Ejemplo de pseudo implementaci√≥n en Python del algoritmo de generaci√≥n.

    def generate(n_rows, n_cols, ratio, diff):
    hidato_solved = generate_hidato_solved(n_rows, n_cols, ratio)
    max_elem = n_rows * n_cols - count_obstacles(hidato_solved)
    cells_for_remove = random_shuffle(filter(lambda cell: get_cell_value(cell) > 1 and get_cell_value(cell) < max_elem, get_cells(hidato_solved)))
    cant_empty = floor(len(cells_for_remove) * diff)
    template = remove_cells(hidato_solved, n_rows, n_cols, cells_for_remove, cant_empty)
    return template


def generate_hidato_solved(n_rows, n_cols, ratio):
    pseudo_template = generate_random_pseudo_template(n_rows, n_cols, ratio)
    limit = n_rows * n_cols - count_obstacles(pseudo_template)
    solves = solve(pseudo_template, 0, limit)
    if solves:
        return solves[0]
    else:
        return generate_hidato_solved(n_rows, n_cols, ratio)


def generate_random_pseudo_template(n_rows, n_cols, ratio):
    cant_obs = floor(n_rows * n_cols * ratio)
    blank_template = create_black_template(n_rows, n_cols)
    cells_for_remove = take(cant_obs, random_shuffle(get_cells(blank_template)))
    dark_template = reduce(lambda acc, cell: update_cell_in_table(acc, update_value_in_cell(-1, cell)), cells_for_remove, blank_template)
    pseudo_template = update_cell_in_table(dark_template, update_value_in_cell(1, select_random_cell(dark_template)))
    return pseudo_template


def remove_cells(template, n_rows, n_cols, cells_for_remove, cant_empty):
    if not cells_for_remove or cant_empty < 0:
        return template
    else:
        head_cell = cells_for_remove[0]
        tail_cells = cells_for_remove[1:]
        empty = update_value_in_cell(0, head_cell)
        new_template = update_cell_in_table(template, empty)
        limit = n_rows * n_cols - count_obstacles(new_template)
        solves = solve(new_template, 0, limit)
        if len(take(2, solves)) < 2:
            return remove_cells(new_template, n_rows, n_cols, tail_cells, cant_empty - 1)
        else:
            return remove_cells(template, n_rows, n_cols, tail_cells, cant_empty)

    Solucionadorüîó

    La "cara p√∫blica" del solucionador es la funci√≥n solveAll la cual mediante la caracter√≠stica llamada currying de Haskell que permite una aplicaci√≥n parcial de una funci√≥n, en este caso la funci√≥n solveAll realiza una aplicaci√≥n parcial de la funci√≥n solveRecursiveDFS convirti√©ndose la funci√≥n solveAll en una funci√≥n que recibe dos argumentos, una Matrix y una lista de enteros.

    solveAll :: Matrix -> [Int] -> [Matrix]
solveAll m = solveRecursiveDFS m 1 (Cell 0 0 0) (rows m * columns m  - countObstacles m) (buildMap m)

    En el llamado a la funci√≥n solveRecursiveDFS se utilizan dos funciones countObstacles y buildMap.

    La funci√≥n countObstacles calcula la cantidad de celdas con valor igual a -1 auxili√°ndose de la funci√≥n countInMatrix que permite contar la cantidad de celdas con valor igual a un n√∫mero dado. Esta funci√≥n se utiliza para contar la cantidad de casillas que no son obst√°culos.

    countInMatrix :: Int -> Matrix -> Int
countInMatrix val (Matrix _ _ cells) =
                    foldl (\acc cell -> if value cell == val then acc + 1 else acc) 0 cells

countObstacles :: Matrix -> Int
countObstacles = countInMatrix (-1)

    La funci√≥n buildMap genera un mapa o diccionario a partir de las celdas de una matriz, utilizando como llaves los valores de las celdas y como valores las celdas. Este diccionario es utilizado posteriormente para saber si una casilla con un valor determinado se encuentra en el tablero y de ser as√≠ acceder a ella eficientemente.

    buildMap :: Matrix -> Map Int Cell
buildMap ma@(Matrix r c m) = Set.foldl (\acc cell -> Map.insert (value cell) cell acc) Map.empty m

    La funci√≥n solveRecursiveDFS recibe 6 argumentos.

    1. actualMatrix: En la matriz actual, representando una plantilla de Hidato v√°lida con posibilidades de ser un Hidato resuelto.
    2. step: Es el paso del algoritmo y representa el n√∫mero que se est√° intentando rellenar, pero puede existir ya en el tablero.
    3. prevCell: Es la celda que tiene el valor (step -1), ya sea porque estaba en el tablero o porque se rellen√≥, este variable tiene el objetivo de disminuir la complejidad temporal del algoritmo evitando tener que realizar el procesamiento para obtener en cada paso.
    4. obs: Es el n√∫mero de casillas que NO son obst√°culos.
    5. map: Es un diccionario de entero contra celdas, en la explicaci√≥n sobre la funci√≥n buildMap se puede leer m√°s sobre su objetivo.
    6. seeds: Una lista de enteros infinita, su objetivo es ser utilizada como semilla para generar n√∫mero pseudo aleatorios utilizados en el proceso de recorrer las casillas adyacentes haciendo en cada paso del algoritmo en un orden no necesariamente igual.

    La funci√≥n lo que hace es comprobar si se est√° en el √∫ltimo paso (que ya se rellenaron todas las casillas posibles y se lleg√≥ a un Hidato resuelto) o por lo contrario realizar los llamados recursivos buscando las posibles soluciones. (La explicaci√≥n del algoritmo se encuentra en la secci√≥n Algoritmo de Soluci√≥n).

    solveRecursiveDFS :: Matrix -> Int -> Cell -> Int -> Map Int Cell -> [Int] -> [Matrix]
solveRecursiveDFS actualMatrix step prevCell obs map seeds
      | step == obs + 1 = [actualMatrix]
      | otherwise = let toAdd = stepMatrix step actualMatrix prevCell map seeds
                    in concat [ solveRecursiveDFS matrix (step + 1) prevCell obs map (tail seeds) | (matrix, prevCell) <- toAdd ]

    La funci√≥n stepMatrix calcula las posibles matrices o plantillas de Hidato v√°lidas partiendo de una plantilla recibida como argumento y el n√∫mero que tiene que rellenar en la matriz. Esto lo hace utilizando prevCell y map para mejorar la complejidad temporal y realizar menos c√°lculos adem√°s de la utilizaci√≥n de otras funciones como getAdjacents e isAdjacent.

    stepMatrix :: Int -> Matrix -> Cell -> Map Int Cell -> [Int] -> [(Matrix, Cell)]
stepMatrix step m@(Matrix rs cs ma) prevCell map seeds = if Map.notMember step map
      then [
            newMatrix | cell <- getAdjacents prevCell rs cs step seeds,
            let actCell = Set.elemAt (Set.findIndex cell ma) ma,
            value actCell == 0,
            let newMatrix = (Matrix rs cs (Set.insert cell ma), cell)
      ]
      else  let actCell = map Map.! step
            in [newMatrix | isAdjacent actCell prevCell || value actCell == 1, let newMatrix = (m, actCell)]

getAdjacents :: Cell -> Int -> Int -> Int -> [Int] -> [Cell]
getAdjacents (Cell r c v) rs cs s seeds = [
        Cell nr nc s |
        dr <- genShuffle seeds [-1, 0, 1],
        dc <- genShuffle seeds [-1, 0, 1],
        let (nr, nc) = (r + dr, c + dc),
        nr > 0,
        nr <= rs,
        nc > 0,
        nc <= cs,
        not $ dr == 0 && dc == 0
    ]

isAdjacent :: Cell -> Cell -> Bool
isAdjacent (Cell f1 c1 v1) (Cell f2 c2 v2)
            | Cell f1 c1 v1 == Cell f2 c2 v2           = False
            | abs (f1 - f2) >= 2 || abs (c1 - c2) >= 2 = False
            | v1 == (-1) || v2 == (-1)                 = False
            | otherwise                                = True

    Generadorüîó

    La "cara p√∫blica" del generador es la funci√≥n generateGame, esta recibe la cantidad de filas y columnas, el porcentaje de obst√°culos, la dificultad y un l√≠mite de tiempo que tendr√° en generador para generar la plantilla de Hidato v√°lida (m√°s adelante se explicar√° con m√°s detalles este l√≠mite de tiempo y porque es necesario). La funci√≥n devuelve una tupla, en la primera componente significa si se puedo generar la plantilla y en la segunda componente la plantilla generada (notar que si la primera componente es igual a False, el valor de la segunda componente no tiene significado).

    generateGame :: Int -> Int -> Float -> Difficulty -> Int -> IO (Bool, Matrix)
generateGame rn cn ratio dif to = do
      maybeMatrix <- timeout to $ generateRandomGame rn cn ratio
      if isNothing maybeMatrix then do
            return (False, blankMatrix 1 1)
      else do
            let solution = maybe (blankMatrix rn cn) id maybeMatrix
            let maxElem = rn * cn - countObstacles solution
            let setForRemove = Set.filter (\x -> let v = value x in v > 1 && v < maxElem) (matrix solution) :: Set Cell
            randomCells <- genRCellFromSet setForRemove
            let total = Set.size setForRemove
            let cant_empty = floor $ fromIntegral total * emptyRatio dif
            seed <- randomIO :: IO Int
            let gen = mkStdGen seed
            let seeds = randoms gen :: [Int]
            let game = removeCells solution randomCells cant_empty seeds
            return (True, game)

    La funci√≥n generateGame se auxilia de la funci√≥n generateRandomGame para obtener un Hidato resuelto. Para realizar esta operaci√≥n se utiliza un tiempo l√≠mite, debido a la naturaleza aleatoria de generateRandomGame (no se asegura que termine, porque puede dependiendo de los par√°metros demorar mucho o incluso con una baj√≠sima probabilidad realizar un ciclo infinito).

    En caso de sobrepasar el l√≠mite de tiempo es devuelto una tupla con la primera componente en False. En caso de no sobrepasar el tiempo, se realiza un proceso de "eliminaci√≥n" (poner con valor cero varias celdas) comprobando en cada paso que al hacerlo se mantenga la unicidad de la soluci√≥n de esa plantilla (que los posibles Hidato resueltos es solamente uno).

    generateRandomGame :: Int -> Int -> Float -> IO Matrix
generateRandomGame rn cn ratio = do
      maybeTemplate <- timeout 1000000 $ generateRandom rn cn ratio
      if isNothing maybeTemplate then
            generateRandomGame rn cn ratio
      else do
            let template = maybe (blankMatrix rn cn) id maybeTemplate
            seed <- randomIO :: IO Int
            let gen = mkStdGen seed
            let seeds = randoms gen :: [Int]
            let solutions = solveAll template seeds
            if null solutions then
                  generateRandomGame rn cn ratio
            else
              return $ head solutions

    El orden de las casillas escogidas para eliminar se realiza con una permutaci√≥n aleatoria del conjunto de casillas disponibles para eliminar (las que tengan como valor mayor que 1 y menor que el m√°ximo). Luego utilizando la funci√≥n removeCells que lo que realiza es recorrer la permutaci√≥n aleatoria y comprobar si eliminando la casilla i-√©sima se mantiene la unicidad, en caso de ser as√≠ se elimina y se pasa la siguiente casilla dentro de la permutaci√≥n aleatoria.

    removeCells :: Matrix -> [Cell] -> Int -> [Int] -> Matrix
removeCells sol@(Matrix rn cn cs) cells n seeds =
      if    null cells || n < 0 then
            sol
      else
